# LogRep2 ビルド手順書

## 1. 前提環境

- Windows 10／11 x64
- .NET 8 SDK
- PowerShell

コマンドは`logRep2`フォルダーで実行する。

```powershell
cd C:\work\environments\logRep_r\logRep2
```

本書では、用途を次のように区別する。

| 用途 | 構成 | 用途の説明 |
| --- | --- | --- |
| デバッグ用 | `Debug` | 開発中の動作確認とデバッガー接続用 |
| テスト用 | `Release` | 受け入れ確認や配布前確認に近い最適化状態での試験用。障害調査用PDBは残す |

## 2. 事前復元

初回、依存関係の変更後、または`obj`フォルダーを削除した後に実行する。

```powershell
dotnet restore src/LogRep2.App/LogRep2.App.csproj -r win-x64
```

以降の例では復元済みとして`--no-restore`を指定している。復元していない場合は、各コマンドから`--no-restore`を外す。

## 3. デバッグ用ビルド

### 3.1 ランタイム同梱版

.NETランタイムが入っていないWindows x64 PCでも実行できる自己完結型である。出力サイズは大きくなる。

```powershell
dotnet publish src/LogRep2.App/LogRep2.App.csproj `
  -c Debug `
  -r win-x64 `
  --self-contained true `
  --no-restore `
  -p:PublishSingleFile=true `
  -p:PublishTrimmed=false `
  -p:PublishReadyToRun=false `
  -p:DebugType=portable `
  -p:DebugSymbols=true `
  -o artifacts/debug/win-x64-self-contained
```

主な出力:

```text
artifacts\debug\win-x64-self-contained\LogRep2.exe
artifacts\debug\win-x64-self-contained\LogRep2.pdb
```

### 3.2 ランタイムなし版

実行先に.NET 8 Desktop Runtime x64が必要なフレームワーク依存型である。自己完結型より出力サイズが小さい。

```powershell
dotnet publish src/LogRep2.App/LogRep2.App.csproj `
  -c Debug `
  -r win-x64 `
  --self-contained false `
  --no-restore `
  -p:PublishSingleFile=true `
  -p:PublishTrimmed=false `
  -p:PublishReadyToRun=false `
  -p:DebugType=portable `
  -p:DebugSymbols=true `
  -o artifacts/debug/win-x64-framework-dependent
```

主な出力:

```text
artifacts\debug\win-x64-framework-dependent\LogRep2.exe
artifacts\debug\win-x64-framework-dependent\LogRep2.pdb
```

## 4. テスト用ビルド

### 4.1 ランタイム同梱版

Release最適化を有効にした自己完結型である。ランタイム未導入PCを含む受け入れ確認に使用する。

```powershell
dotnet publish src/LogRep2.App/LogRep2.App.csproj `
  -c Release `
  -r win-x64 `
  --self-contained true `
  --no-restore `
  -p:PublishSingleFile=true `
  -p:PublishTrimmed=false `
  -p:PublishReadyToRun=false `
  -p:DebugType=portable `
  -p:DebugSymbols=true `

```

主な出力:

```text

artifacts\test\win-x64-self-contained\LogRep2.pdb
```

### 4.2 ランタイムなし版

Release最適化を有効にしたフレームワーク依存型である。実行先に.NET 8 Desktop Runtime x64が必要になる。

artifacts\test\win-x64-framework-dependent\LogRep2.pdb

## 5. テスト実行

ビルド成果物の作成とは別に、自動テストをDebug／Releaseの両方で実行する。

```powershell
dotnet test LogRep2.sln -c Debug --no-restore
dotnet test LogRep2.sln -c Release --no-restore
```

フォーマットも確認する。

```powershell
dotnet format LogRep2.sln --verify-no-changes --no-restore
```

## 6. 起動確認

例として、テスト用ランタイム同梱版を起動する。

```powershell
.\artifacts\test\win-x64-self-contained\LogRep2.exe
```

CLIの確認:

```powershell
.\artifacts\test\win-x64-self-contained\LogRep2.exe help
.\artifacts\test\win-x64-self-contained\LogRep2.exe config path
```

`config path`が、起動した`LogRep2.exe`と同じフォルダーの`LogRep2.settings.json`を示すことを確認する。設定はユーザーホーム、AppData、レジストリへ作成しない。

## 7. ランタイム同梱有無の選択基準

| 項目 | ランタイム同梱版 | ランタイムなし版 |
| --- | --- | --- |
| `--self-contained` | `true` | `false` |
| 実行先の.NET 8 Desktop Runtime | 不要 | 必要 |
| ファイルサイズ | 大きい | 小さい |
| 未導入PCでの受け入れ確認 | 適する | 適さない |
| 開発PC内での短時間確認 | 利用可能 | 適する |

正式配布用は、デバッグシンボルを含めない既存の`win-x64` publish profileを使用する。

```powershell
dotnet publish src/LogRep2.App/LogRep2.App.csproj `
  --no-restore `
  -p:PublishProfile=win-x64 `
  -o artifacts/LogRep2-win-x64
```

## 8. 注意事項

- `PublishTrimmed`はWPFやリフレクション利用箇所への影響を避けるため、すべて`false`とする。
- Debug用・テスト用のPDBにはソース情報が含まれるため、正式配布物へ入れない。
- ランタイムなし版で起動できない場合は、Microsoftの.NET 8 Desktop Runtime x64を実行先へ導入する。
- 各出力先には設定ファイルが作られるため、異なるビルドを同じ出力フォルダーへ上書きしない。
- `Program Files`など一般ユーザーが書き込めない場所から起動しない。
